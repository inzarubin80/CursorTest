# ТЗ: приложение «Заметки» (React, хранение в браузере)

## 1. Цель

Веб-приложение для создания, просмотра, редактирования и удаления заметок. Данные хранятся только в браузере пользователя (без бэкенда).

---

## 2. Стек

- **React** (Vite или Create React App)
- **Хранение данных:** `localStorage` (при необходимости позже — IndexedDB для больших объёмов)
- Стили: на усмотрение (CSS-модули, Tailwind, обычный CSS)

---

## 3. Функциональные требования

### 3.1 Заметка

- **Поля:** заголовок, текст, дата создания (опционально — дата последнего изменения).
- **Операции:** создание, просмотр, редактирование, удаление.

### 3.2 Список заметок

- Отображение списка заметок (заголовок + превью текста или дата).
- Сортировка: по дате создания (новые сверху или снизу — зафиксировать в ТЗ).
- Поиск по заголовку и/или по тексту заметки (опционально).

### 3.3 Хранение

- При каждом изменении (добавление, редактирование, удаление) данные сохраняются в `localStorage`.
- При загрузке приложения данные поднимаются из `localStorage` и отображаются (никакого сервера).

### 3.4 Интерфейс

- Список заметок слева или сверху; справа/снизу — просмотр/редактирование выбранной заметки (или одна колонка: список → при клике открывается форма/карточка).
- Кнопки/действия: «Новая заметка», «Редактировать», «Удалить», «Сохранить» (если редактирование в отдельном режиме).
- Подтверждение перед удалением (опционально, но желательно указать в ТЗ).

---

## 4. Структура данных

```
Заметка (Note):
  id: string (uuid или Date.now().toString())
  title: string
  body: string
  createdAt: number (timestamp)
  updatedAt?: number (timestamp, опционально)

localStorage ключ: "notes-app-data"
Значение: JSON-массив заметок (Note[])
```

---

## 5. Нефункциональные требования

- Адаптивная вёрстка: удобно на телефоне и на десктопе.
- Работа в современных браузерах (Chrome, Firefox, Safari, Edge).
- Офлайн не обязателен (при желании позже — Service Worker).

---

## 6. Этапы разработки

1. Инициализация React-проекта, базовая разметка.
2. Модель данных: тип/интерфейс заметки, чтение/запись в `localStorage`.
3. Список заметок: отображение, выбор заметки, сортировка.
4. Просмотр и редактирование: форма/поля заголовка и текста, сохранение.
5. Создание новой заметки и удаление (с подтверждением при необходимости).
6. Поиск по заметкам (если нужен).
7. Базовая стилизация и адаптивность.

---

## 7. Возможные расширения (позже)

- Теги/категории у заметок.
- Папки/разделы.
- Экспорт/импорт (JSON или файл).
- Тёмная тема.
- Rich-text или Markdown в теле заметки.
- Переход на IndexedDB при росте объёма данных.
