# Документирующие комментарии 1С (BSL)

Источник: Система стандартов и методик разработки конфигураций (v8std), раздел «Описание процедур и функций» (ИТС), 1C:EDT — «Документирующие комментарии».

## Назначение

Документирующий комментарий — это комментарий к собственной процедуре или функции, написанный по определённым правилам. В нём перечисляются параметры, их типы и назначение, тип и назначение возвращаемого значения (для функций).

## Структура комментария

1. **Описание** — одна или несколько строк с кратким описанием назначения метода.
2. **Параметры:** — ключевое слово с двоеточием. Затем для каждого параметра строка вида:
   - `ИмяПараметра - Тип - описание назначения.`
   - Строка параметра начинается с отступа (пробелы после `//`).
3. **Возвращаемое значение:** (только для функций) — тип и назначение возвращаемого значения.

## Пример для функции

```bsl
// Вычисляет сумму двух числовых значений.
//
// Параметры:
//   Значение1 - Число - первое слагаемое.
//   Значение2 - Число - второе слагаемое.
//
// Возвращаемое значение:
//   Число - сумма Значение1 и Значение2.
Функция Сумма(Значение1, Значение2)
	Возврат Значение1 + Значение2;
КонецФункции
```

## Пример для процедуры

```bsl
// Выполняет проверку заполнения реквизитов.
//
// Параметры:
//   Данные - Структура - проверяемые данные.
//   ПроверяемыеРеквизиты - Массив из Строка - имена реквизитов.
//   Отказ - Булево - признак отказа (устанавливается по ссылке).
Процедура ПроверитьЗаполнение(Данные, ПроверяемыеРеквизиты, Отказ)
	Отказ = Ложь;
КонецПроцедуры
```

## Типичные ошибки

- Забывают поставить двоеточие после ключевого слова («Параметры» → «Параметры:»).
- Не ставят отступ у строк параметров (после `//` должны быть пробелы перед именем параметра).
- Формат строки параметра: именно «Имя - Тип - Описание», разделитель — « - » (пробел-дефис-пробел).
- Для структур, таблиц значений, деревьев значений можно описывать поля/колонки (см. v8std, раздел 641).
